# UrbanEV-v2: MATSim EV Simulation for Maryland + DC

## Project Overview

Agent-based electric vehicle simulation using MATSim with the UrbanEV-v2 extension.
Simulates ~116K EV agents across the Maryland + DC road network to study charging
infrastructure utilization, range anxiety, and energy demand.

## Coordinate Reference System

**All spatial data uses EPSG:26985 (NAD83 / Maryland State Plane).**
The MATSim network, charger locations, agent coordinates, and through stations are
all in this CRS. AFDC charger data (WGS84) is transformed during conversion.

## Key Data Facts

### Synthetic Population CSV (6.6 GB)
- **NEVER load fully into memory.** Stream with csv.DictReader or pandas chunksize.
- File is sorted by person_id (monotonically increasing). Safe to flush-on-boundary.
- 20,285,076 rows, 63 columns, ~6.05M unique persons.
- Only `assigned_ev=1` persons (~116K) are included in MATSim outputs.
- Coordinates already in EPSG:26985 — no transformation needed.
- Time format: integer HHMM (e.g., 1430 = 14:30). Convert: `h*3600 + m*60` for MATSim seconds.
- Some o_x/d_x values are 0.0 — these are external trips resolved via through stations.

### Column Code Mappings (from RTS codebook)

**o_activity / d_activity:**
1=Home, 2=Work, 3=Volunteer, 4=School, 5=Shopping, 6=Meal(quick),
7=Meal, 8=Gas, 9=Healthcare, 10=Errand, 11=Socialize, 12=Civic/Religious,
13=Exercise, 14=Recreation, 15=Entertainment, 16=Drop-off/Pick-up, 18=Other.
MATSim mapping: 1→"home", 2→"work", 4→"school", all others→"other".

**travel_mode:**
1=Walk, 2=Bike, 3=Motorcycle, 4=Auto(driver), 5=Auto(passenger),
6=SchoolBus, 7=Rail, 8=Bus, 9=PrivateBus, 10=Paratransit,
11=Taxi, 12=Uber/Lyft, 15=Other.
MATSim mapping: 4→"car", all others→"walk" (teleported).
**WARNING:** `is_car_trip=1` includes Walk/Bike/Motorcycle. True car-driver = travel_mode==4 only.

**fueltype (synthetic pop encoding, NOT codebook order):**
0=N/A(non-motorized), 1=Gasoline, 2=Diesel, 3=PHEV, 4=BEV, 5=HEV.

**hh_income_detailed (MPO 10-bracket, 0-indexed):**
0=<$10K, 1=$10-15K, 2=$15-25K, 3=$25-35K, 4=$35-50K,
5=$50-75K, 6=$75-100K, 7=$100-150K, 8=$150-200K, 9=$200K+.
Midpoints: [7500, 12500, 20000, 30000, 42500, 62500, 87500, 125000, 175000, 250000].

**home_type (MPO_HOME_TYPE):**
0=Missing, 1=SFD, 2=SFA/townhouse, 3=2-9 unit apt,
4=10-49 unit apt, 5=50+ unit apt, 6=Mobile home.

**home_ownership (MPO_HOME_OWNERSHIP):**
0=Missing, 1=Own/mortgage, 2=Rent, 3=Job/military, 4=Family/friend.

**employment_status:**
0=Worker, 1=Retired, 2=Volunteer, 3=Homemaker, 4=Unemployed(seeking),
5=Unemployed(not seeking), 6=Student, 7=Disabled, 8=Child(synthetic), 9=N/A(synthetic).

**gender:** 1=Female, 2=Male, 0/3=Other/not reported (synthetic additions).

**license:** 1=Yes, 2=No, 0=Under 15, 3=Missing.

### MATSim Network
- 4,909,987 nodes, 9,831,519 links (OSM-derived, fully connected, no dead ends).
- Bounding box: X 183,658–570,205, Y 32,787–249,470 (EPSG:26985).
- Extends beyond MD/DC state boundaries.

### Chargers
- 1,958 entries: 1,607 L2 + 283 DCFC + 68 DCFC_TESLA.
- Top networks: ChargePoint (737), Blink (425), SWTCH (343).
- Power: L2=7.2kW, DCFC varies by network (50–150kW).

### Through Stations
- 14 highway boundary crossings for external trip resolution.
- Top usage: TS_US301_N (1,945 trips), TS_I83_N (1,256), TS_DC_NE (1,026).
- State coverage: VA(51), PA(42), DE(10), WV(54), NJ(34), NY(36), DC(11).

## Derived Attribute Rules

**homeChargerPower (kW):**
- SFD + own + income>=6: 7.2
- SFD + own + income 4-5: 60%→7.2, 40%→1.4
- SFD + own + income<4: 1.4
- SFD + rent + income>=5: 1.4, else 0
- SFA + own + income>=5: 1.4, else 0
- Apartments: 0
- Mobile + own: 1.4

**betaMoney:** `-6.0 * (62500 / income_midpoint)`
**valueOfTime:** `(income/2080) * 0.5`, reduced by 0.6x for non-workers.

## Scripts

### convert_afdc_to_chargers_xml.py
Reads AFDC CSVs → chargers.xml + chargers_metadata.csv.
Run: `python3 "Input Files/convert_afdc_to_chargers_xml.py"`

### convert_synpop_to_matsim.py
Streams 6.6GB CSV → plans.xml.gz + electric_vehicles.xml + vehicletypes.xml + summary.csv.
Requires through_stations.csv in same directory.
Run: `python3 "Input Files/convert_synpop_to_matsim.py"` (~2.5 min)

### generate_docs.py
Regenerates prompt_log.pdf and pipeline_guide.pdf.
Run: `python3 generate_docs.py`
**Update this script when adding new prompts/stages, then re-run.**

### analyze_equity_corridors.py
Post-processing script for simulation outputs.
Run: `python3 analyze_equity_corridors.py --output-dir scenarios/maryland/output/maryland_ev_enhanced/`
Outputs: equity_analysis.html, corridor_soc_profiles.html, charger_utilization.html + CSVs.

### Integration Test (generate + run + validate)
A full round-trip test verifying framework behaviour with 50 agents, 15 iterations:
```bash
bash run_integration_test.sh          # full pipeline (build + sim + validate)
bash run_integration_test.sh --skip-build  # skip Maven rebuild
bash run_integration_test.sh --skip-sim    # validate existing output
```
All 8 behavioural assertions pass on JDK 23. See `test_scenario/` for inputs.

**JDK 23 notes** (Maven enforcer allows JDK 11-17; override with `-Denforcer.skip=true`):
- Build: `mvn clean package -DskipTests -Denforcer.skip=true`
- Run: requires `--add-opens java.base/java.lang=ALL-UNNAMED` etc. for Guice CGLIB
- DTD fetching for `vehicleDefinitions_v1.0.dtd` fails (matsim.org returns invalid response);
  use vehicleDefinitions v2.0 schema (`xmlns="http://www.matsim.org/files/dtd"`) instead.
- Local DTD copies (chargers, electric_vehicles) are in `test_scenario/dtd/` to avoid web fetches.

## Conventions
- Random seed: 42 (for reproducibility across runs)
- All XML uses UTF-8 encoding
- Plans file is gzipped (compresslevel=6)
- Person IDs match across plans, electric_vehicles, and summary CSV
- Vehicle type keys: lowercase, spaces→underscores (e.g., "model_y", "f_150_lightning")

## MATSim Config (scenarios/maryland/config.xml)

- **13 modules:** global, network, plans, vehicles, transit, controler, qsim,
  parallelEventHandling, planCalcScore, strategy, urban_ev, subtourModeChoice,
  TimeAllocationMutator
- **60 iterations**, events written every 10
- **flowCapacityFactor=0.02**, storageCapacityFactor=0.06 (EV-only, 116K of ~6M pop)
- **QSim endTime=168:00:00** (7-day simulation window)
- **14 activity types:** home, work, school, other + 4 charging + 4 charging failed
  + car interaction + pt interaction
- **2 subpopulations:** nonCriticalSOC (SelectExpBeta 0.6 / ChangeCharging 0.2 /
  InsertEnRoute 0.2), criticalSOC (ChangeCharging 0.4 / InsertEnRoute 0.6)
- **38 urban_ev params** including power-tier pricing, VoT, smart charging, en-route
- **Pricing (MD rates):** home $0.13/kWh, work free, L2 $0.25/kWh, DCFC $0.48/kWh
- Innovation disabled after 80% of iterations (fractionOfIterationsToDisableInnovation=0.80)
- Input file paths use `../../Input Files/` relative references from scenarios/maryland/
- Output: `output/maryland_ev_enhanced/`

## Charger Type Compatibility

All EVs and chargers must use matching type strings for `findBestCharger()` to work:
- **Vehicle charger_types:** Tesla → `L1,L2,DCFC,DCFC_TESLA`, non-Tesla → `L1,L2,DCFC`
- **Public chargers (chargers.xml):** types `L2`, `DCFC`, `DCFC_TESLA`
- **Private chargers (Java-generated):** Must use `L1` (power < 3kW) or `L2` (power >= 3kW)
  - **REQUIRES Java fix:** `MobsimScopeEventHandling.addPrivateCharger()` currently uses
    `DEFAULT_CHARGER_TYPE = "default"` — must be changed to derive type from power level

## Known Limitations
- External trip portions (outside MD/DC) are not modeled for energy consumption.
  Agents enter/exit at through stations; only in-network SoC is tracked.
- ~49% of external trips already had coordinates (nearby VA/DE within network extent);
  only trips with (0,0) coords needed through-station substitution.
- Battery initial_soc is randomized 40-80% of capacity — not calibrated to real data.
- Home charger assignment uses probabilistic rules, not actual charger registry data.
